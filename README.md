# Generative AI Game Engine: Independent Study 🎮

## 📖 Overview
This project explores the intersection of **Generative AI** and **Game Development** by comparing the efficiency and flexibility of three distinct text-to-game generation methods. The study evaluates how Large Language Models (LLMs) can be used to either modify existing game logic or generate entirely new interactive environments through natural language processing.

## 📁 Repository Structure

### 🛠️ Hardcoded Games
*This folder represents the baseline "controlled" environment where logic is pre-defined.*
* **`TextGame.py`**: The main driver for the hard-coded variant. This serves as the performance and stability benchmark for the engine.
* **`Planet.py`**: A class-based script used to instantiate procedurally generated planet objects.
* **`Orbits.py`**: Handles the physics engine responsible for calculating and displaying spacecraft trajectories.
* **`Images.py`**: A utility script for loading, scaling, and assigning assets across the game environment.

### 🤖 Dynamically Coded Games
*This folder contains the core research implementation using OpenAI's API.*
* **`DynamicGame.py`**: The primary research tool. It interfaces with OpenAI to generate game code that remains **mutable**; users can edit specific parameters in real-time via text input.
* **`TextInput.py`**: A direct interface with OpenAI. While it can generate complex game logic from scratch, it differs from the dynamic variant as it is not designed for real-time mutation after generation.
* **`BaseGame.py`**: A default "Pong" style test environment used to verify API connectivity and basic rendering.
* **`ModifiedDynamicGame.py`**: A temporary file generated by `DynamicGame.py` to store AI-edited logic.
* **`ModifiedTextGame.py`**: A temporary file generated by `TextInput.py` during the generation process.
* **`Invalid.py`**: A fallback script that triggers an error screen if the AI-generated code fails to compile or run.

## 🛠️ Tech Stack & Requirements
* **Language:** Python 3
* **Libraries:** * `pygame`: Core rendering and game loop.
    * `openai`: Integration with GPT-based models.
    * `numpy`, `math`, `random`: Physics and procedural generation.
* **Environment:** Python IDE (VS Code/PyCharm) or terminal.

## 💻 How to Use

### Modifying Variables (`DynamicGame.py` & `TextInput.py`)
To modify a game parameter using the AI interface, follow this specific syntax:
1. Ensure the target variable in the source code is written in **ALL CAPS** (e.g., `GRAVITY = 10`).
2. In the text input box, type the **VARIABLE_NAME** followed by the new integer value.

### Creating New Games (`TextInput.py`)
* To initiate the generation of a completely new game concept, type the keyword **`create`** into the text box followed by your prompt description.
